BLAST hits

No BLAST hits found for {{contig}}.

This can have several causes:

  • NCBI CPU throttling — when using remote BLAST, NCBI may silently reject queries for large contigs against the full nt database if too many requests originate from the same IP. The job returns a READY status but with zero hits.
  • E-value threshold — the configured e-value may be too stringent for this contig.
  • Organism not in the searched database — if entrez_query is set, the organism may not be represented in the filtered subset.
  • Genuine absence — the contig sequence may not match anything in the database (novel organism, contamination, assembly artefact).

Workarounds: restrict the search with entrez_query: 'Bacteria[Organism]' and/or switch to remote_db: 'refseq_genomic' to reduce NCBI load (see the LORA README). For reliable results on large contigs, consider downloading a local nt database (~270 GB) and running BLAST locally with blast.remote: false.
